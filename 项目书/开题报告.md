# 开题报告

## 摘要

本报告研究 ReLU 全连接神经网络在 $L_\infty$ 扰动下的鲁棒性验证，选取 alpha-beta-CROWN 作为主平台开展调研与实验。前期已完成环境搭建、命令行自检和基础复现实验，并建立“配置执行-日志归档-结果汇总-图表生成”的流程。MNIST 结果表明，样本规模从 20 扩展到 100 后，策略差异更明显，kfsb 在验证率与时间成本上相对更优。后续将按“主线 A（分支策略优化）+ 辅线 B（扰动半径扫描）”推进，输出可复核的阶段结论与汇报材料。

关键词：神经网络鲁棒性验证；ReLU；alpha-beta-CROWN；Branch-and-Bound；软件质量保障

## 1. 研究意义

深度神经网络已在视觉识别、医疗辅助、智能控制等场景中广泛应用，但其在微小输入扰动下可能出现预测翻转。传统模型评估主要依赖自然准确率，无法回答“在可接受扰动范围内模型是否稳定”这一质量保障核心问题。对于课程所关注的软件质量与可靠性主题，神经网络鲁棒性验证是一个具有代表性的“从经验评估走向可证明质量”的研究方向。

本课题聚焦“ReLU 全连接神经网络在 $L_\infty$ 扰动下的形式化鲁棒性验证”，研究意义体现在以下四个方面。

1. 理论意义：将鲁棒性问题表达为全称约束判定问题。对于样本 $x$、标签 $y$ 与扰动半径 $\epsilon$，验证目标可写为：对于所有满足 $\|x' - x\|_\infty \le \epsilon$ 的输入 $x'$，均有 $f_y(x') - f_j(x') > 0$（$j \ne y$）。该表述将“未找到反例”与“可证明安全”严格区分。
2. 方法意义：比较不完整验证与完整验证在准确性、可扩展性、计算代价上的权衡关系，为不同业务场景下验证策略选型提供依据。
3. 工程意义：形成一套可复现的验证流水线，包括环境搭建、配置管理、日志归档、指标汇总、图表生成与报告输出，可复用于后续模型质量评估任务。
4. 教学意义：与课程第二部分“搭建、分析、改进、实验、汇报”的要求强一致，能完整覆盖工具链实践与实证分析能力。

结合课程评分标准与项目周期，本课题采用“中档目标”策略，即“工具解析 + 复现实验 + 对比结论”，不将源码级算法创新作为硬性目标，优先保证过程可复核、结果可解释、结论可交付。

## 2. 相关现状调研

### 2.1 研究问题与主流技术路线

神经网络鲁棒性验证的核心问题是：对于给定输入样本 $x$ 与扰动半径 $\epsilon$，是否对所有满足 $\|x' - x\|_\infty \le \epsilon$ 的输入 $x'$ 都保持分类性质不变。

当前主流方法大致分为三类：

1. 线性松弛/边界传播：代表方法 CROWN、alpha-CROWN，特点是速度快、扩展性好，但在复杂样本上可能难以直接证明。
2. Branch-and-Bound 完整验证：代表方法 beta-CROWN，特点是通过分支细化提升证明能力，能在更多样本上得到确定结论。
3. SMT/MIP 精确求解：代表工具 Marabou、MIP 求解器，理论精确性强，但在大规模模型上成本较高。

从研究趋势看，近几年验证方法逐渐从“高精度但低扩展”的路线向“松弛传播 + 分支定界”的混合路线收敛。其核心原因是：

1. 图像任务输入维度高，纯 SMT/MIP 在规模上容易遇到瓶颈。
2. 线性松弛方法具备 GPU 友好性，能显著提升批量样本处理效率。
3. 在松弛不够紧时，引入 BaB 可以逐步收缩不确定区域，兼顾效率与证明能力。

因此，在课程项目里采用该主流路线，既有文献支撑，也有工程可行性。

### 2.2 工具调研结论

结合课程周期、硬件资源和可复现性要求，项目采用如下策略：

1. 主平台：alpha-beta-CROWN。
2. 选型理由：
	- 支持不完整验证与完整验证统一流程，可在同一框架下完成方法对比。
	- 对 ReLU 网络支持成熟，配置范式清晰，便于课程成员分工协作。
	- GPU 加速明显，适合在有限时间内完成多个扰动半径与策略组合实验。
	- 提供 CLI 与 API 两种入口，便于后续从“配置复现”扩展到“程序化批量实验”。
3. 对照思路：保留 Marabou 等 SMT 路线作为调研背景，不作为本阶段主执行平台。

为便于课程报告表达，当前工具比较定位如下：

1. alpha-beta-CROWN：主实验平台，负责产出可规模化对比结果。
2. Marabou：方法学对照参照，用于说明 SMT 方向在解释性与完整性上的优势及其规模限制。
3. MIP 路径：在主平台内作为可选精确增强手段，用于解释“证明能力与计算成本”的典型权衡。

### 2.3 已完成调研支撑

已完成文献与工具层面的初步归纳，形成“ReLU 分段线性 + bound propagation + BaB”的主线认识，并结合仓库代码结构识别了入口、核心求解层、启发式策略层与实验配置层，为后续实验设计提供了工程依据。

已完成的调研成果可直接支撑后续实验设计：

1. 指标定义统一：以 verified accuracy（VRA）、timeout 数量、mean/max time 为主。
2. 对比维度明确：固定样本集与模型，控制变量为策略与扰动半径。
3. 证据链要求明确：每个结论均需对应配置、命令、日志与汇总表，不做不可追溯结论。

## 3. 所选择工具初步搭建情况

### 3.1 搭建环境与依赖状态

当前项目已在本地完成可运行搭建，主要状态如下：

1. 操作系统：Linux（WSL2）。
2. GPU：NVIDIA GeForce RTX 4060 Laptop GPU，可用于加速验证。
3. 环境管理：Conda 环境 alpha-beta-crown 已创建并可用。
4. 关键依赖：torch、onnx、onnxruntime、pyyaml、gurobipy 已可导入。

此外，已补齐可视化依赖，用于将实验结果自动生成图表，支持课堂汇报材料快速更新。

### 3.2 功能自检与结果

已完成三层自检：

1. 入口自检：abcrown.py --help 正常输出。
2. 最小样本自检：mnist_baseline 配置在 1 样本上跑通，结果为 safe-incomplete。
3. 批量样本复现：
	- 20 样本下三策略均为 100% verified acc。
	- 100 样本下出现差异，kfsb 在验证率与耗时上相对更优。

当前已完成的关键实验结论（固定 $\epsilon=0.02$，MNIST 0-100）：

1. baseline：verified acc 91.0%，timeout 9。
2. auto：verified acc 91.0%，timeout 9，平均耗时最高。
3. kfsb：verified acc 92.0%，timeout 8，平均耗时最低。

该结果表明，小样本（0-20）下策略差异不明显，而扩展到 0-100 后差异被放大，验证了“扩大样本规模是必要步骤”的判断。

### 3.3 当前实验资产

截至当前，已形成以下可复用资产：

1. 配置资产：baseline、auto、kfsb 三套对比配置。
2. 日志资产：20 样本与 100 样本完整日志。
3. 汇总资产：策略对比 CSV（0-20 与 0-100）。
4. 可视化资产：平均耗时对比图、最大耗时与验证率联合图。
5. 报告资产：阶段验收文档与图表生成脚本。

这些资产可直接复用于扰动半径扩展实验，降低后续工作重复成本。

### 3.4 当前已确认风险

1. 版本偏离官方推荐（当前 Python/torch 版本与官方测试组合不完全一致），存在兼容性波动风险。
2. 从仓库根目录直接调用 API 可能触发路径冲突，当前阶段统一采用 complete_verifier 目录下 CLI 路线规避。
3. 扰动半径增大时 timeout 增多，需通过统一样本与参数设置控制横向可比性。
4. 结果随样本规模变化明显，若仅报告小样本结果，可能导致结论外推失真。

对应规避策略如下：

1. 环境指纹化：每轮实验固定记录 Python、torch、CUDA、GPU 信息。
2. 命令规范化：统一从 complete_verifier 目录执行，避免路径解析差异。
3. 结果双层检查：先小规模估时，再扩展全量，避免无计划超时。
4. 固定随机种子与样本区间，保证横向可比性。

结论：工具已达到“可运行、可复现、可分析”状态，满足中档目标实施条件。

## 4. 进度安排与预期成效

### 4.1 总体目标定位

1. 完成核心工具解析与执行流程梳理。
2. 完成 MNIST ReLU FCNN 的鲁棒性复现实验。
3. 给出多策略、多扰动半径的对比结果与可视化结论。

不将源码级算法改造作为硬性目标，优先保障结果稳定与报告质量。

为保证目标可执行，课题采用“先保底、后扩展”的实施策略：

1. 保底层：完成 3 策略 x 多扰动半径的标准复现实验与图表汇总。
2. 扩展层：主攻分支策略优化 A，并辅以扰动半径扫描 B，形成“方法改进 + 趋势验证”两条结果线。

其中，A 负责给出可对比的改进方案，B 负责给出更完整的图表和趋势解释。

### 4.2 改进目标选择

#### A. 分支策略优化（主线）

这是正式的改进方向，围绕分支策略做细化调整。

1. 调整 `branching.method`、`branching.candidates`、`branching.reduceop`。
2. 对比 `babsr`、`kfsb`、`auto` 在同一模型、同一样本集和同一 epsilon 下的表现。
3. 必要时加入 `input_split.enable` 与 `input_dim_threshold`，判断输入分裂是否能在部分样本上提前收敛。

预期结果：形成分支策略与样本特征之间的对应关系，并用 timeout、mean time、visited nodes、verified acc 进行量化。

#### B. 扰动半径扫描（辅线）

这部分用于补充图表与趋势分析。

1. 固定主策略为 kfsb，扫描多个 epsilon。
2. 形成 VRA-epsilon 和 time-epsilon 曲线。
3. 用曲线解释鲁棒性随扰动增强的衰减过程。

预期结果：补充完整的趋势图，为主线 A 的分析提供上下文。

### 4.4 预期成效

本课题围绕主线 A（分支策略优化）和辅线 B（扰动半径扫描）设定可比对结果，核心目标如下：

| 结果线 | 对比对象 | 预期变化 | 关键指标 |
| --- | --- | --- | --- |
| A 主线 | kfsb / babsr / auto | 更优配置应减少 timeout 与 mean time，并尽量保持或提升 verified acc | verified acc, timeout, mean/max time, visited nodes |
| B 辅线 | 同一策略下不同 epsilon | epsilon 增大时 verified acc 下降、timeout 与耗时上升 | VRA, timeout, mean/max time |
| A+B 联合 | 策略 x epsilon | 更优策略在多个 epsilon 下保持更稳定表现 | 稳定性、趋势一致性 |

验收口径：

1. 实验覆盖不少于 8 组（目标 12 组：3 策略 x 4 半径）。
2. 每组统一统计 verified acc、timeout、mean time、max time（A 组额外统计 visited nodes）。
3. 输出至少 2 张可直接用于汇报的图表与 1 份对比 CSV。
4. 最终报告至少给出 3 条定量结论，且均可回溯到日志与图表。

该成效定义可直接用于老师验收：既能看见改进效果，也能检查结果可复核性。

### 4.5 预期失败场景与应对预案

1. 场景 A：高半径下超时激增，导致实验周期失控。

应对：优先完成低半径点与核心策略组合，保留高半径为扩展项，并在报告中说明资源约束。

2. 场景 B：不同策略差异不显著，难以形成结论。

应对：扩大样本规模并增加超时与最大耗时指标，避免仅看平均值。

3. 场景 C：环境升级或依赖漂移导致复现失败。

应对：固定 conda 环境与版本信息，保留可回滚配置与日志证据。

通过以上预案，项目可在不增加过多复杂度的前提下维持中档目标的稳定达成。

### 4.6 下一步执行规划

后续工作进入改进验证阶段，执行顺序如下：

1. 完成 A 的配置收敛，固定模型、样本区间与 epsilon，形成 3 组可对比的分支策略结果。
2. 补充 B 的图表支撑，围绕多个 epsilon 生成 VRA-epsilon 和耗时曲线。
3. 结合 A+B 的结果给出最终解释，说明策略差异在不同扰动强度下是否稳定。

对应验收口径如下：

1. A 至少形成 3 组可比较配置，并给出 1 条主结论。
2. B 至少形成 4 个 epsilon 点的趋势图，并给出 1 条辅助结论。
3. 最终报告中明确“主改进方向是 A，B 为辅助验证”。
